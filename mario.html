Begin

This is where I import the character, decoration, scene, and background.

Canvas

A canvas element is selected from the HTML, and a 2D drawing context (context) is established. This allows for graphics to be rendered on the canvas.

Gravity and Player

Gravity pulls the player down by adding vertical velocity. The player has a set position, velocity (for both x and y), and dimensions (width and height).

Sprite Animation

The player sprite is drawn onto the canvas at its current position using the drawImage() method. The sprite uses frame-based animation. Then the update function is called every frame to update the player’s position and velocity. It applies gravity and moves the player according to the velocity.

Platform

This platform class represents static platform in the game that the player can stand on it.

Game and Scroll

Scrolloff keeps track of how far the world scrolled as the player moves to the right. And init() initializes the player, platforms, and other game objects.

Game Loop

animate() is the main game loop. It uses requestAnimationFrame() to update and redraw the game.

Player Movement and Collisions

This code handles user input. When keys are pressed (A for left, D for right, W for jump), the corresponding actions are triggered by changing the player’s velocity or direction. When the player’s position overlaps with a platform, their downward velocity is reset to zero, allowing them to stand on the platform.

Winning and Resettings

If the player has scrolled past a certain point (scrollOff > 10000), the game is won, and a congratulatory message is displayed.
